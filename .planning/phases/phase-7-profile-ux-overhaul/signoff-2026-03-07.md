### P2 — StatCard Visual Redesign

**Files:** `apps/web/src/components/data/StatCard.tsx`, `apps/web/src/components/data/TrendIndicator.tsx`

**BenchmarkSlot interface** (exported from StatCard):
- `localLabel`, `schoolRaw/localRaw/nationalRaw` for proportional bars
- `isPercent` — fixed 0–100 scale for percent metrics; dynamic max scale otherwise
- `displayDecimals` — raw values rounded to match display precision before bar computation (fixes mismatched bar widths when sub-display-precision differences exist)
- `schoolValueFormatted / localValueFormatted / nationalValueFormatted`
- `schoolVsLocalDelta / schoolVsNationalDelta` + formatted equivalents

**BenchmarkBlock** (internal to StatCard):
- sm+: proportional bar chart — school (purple), local (cyan), England (green)
- mobile (<sm): compact text rows with colour dots, no bars
- "This school" row included; "Compared with" heading omitted (cleaner)

**description prop + ⓘ toggle:**
- `description?: string` on StatCard
- ⓘ (Info icon) button appears when description is provided
- Tap/click toggles a plain-English explanation panel below the metric label
- Allows jargon labels to coexist with full explanations without cluttering the card

**TrendIndicator `period` prop:**
- Appends time span to the delta label: `+2.1% · 3yr`
- Passed through from sparkline data length in each section component

### P3 — Benchmark Wiring

**Files:**
- `apps/web/src/features/school-profile/components/AcademicPerformanceSection.tsx`
- `apps/web/src/features/school-profile/components/AttendanceBehaviourSection.tsx`
- `apps/web/src/features/school-profile/components/WorkforceLeadershipSection.tsx`
- `apps/web/src/features/school-profile/SchoolProfileFeature.tsx`
- `apps/web/src/features/school-profile/components/BenchmarkComparisonSection.tsx` (deleted)

All three metric sections accept `benchmarkDashboard: BenchmarkDashboardVM | null`. Each builds a `Map<metricKey, BenchmarkMetricVM>` lookup and converts to `BenchmarkSlot` at card-build time via a local `toBenchmarkSlot()` helper.

`barDecimals(unit)` helper in each section determines display precision:
- `count` / `currency` → 0dp
- `ratio` → 1dp
- `rate` / `score` / `percent` → 2dp

`BenchmarkComparisonSection.tsx` deleted — data is now fully inline. Recoverable from git history if needed for a future compare or print view.

`SchoolDetailsSection` removed from `SchoolProfileFeature` — school administrative detail data is incomplete in the current dataset.

## Known Constraints

- Deprivation domain tile label format ("Decile X / 10") is a temporary hold. The word "decile" is jargon but no agreed plain-English alternative was confirmed during this session. Revisit in P5/P6.
- House price data is at local authority district level — broader than the 1-mile crime radius or LSOA deprivation match. The section subtitle already reflects this.
- Crime radius is configurable in the backend pipeline (default 1 mile / 1609m). The UI displays `crime.radiusMiles` directly from the API — no hardcoding.
