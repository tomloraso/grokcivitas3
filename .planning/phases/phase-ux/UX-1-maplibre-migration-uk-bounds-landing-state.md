## Objective

Replace the raster Leaflet map stack with a UK-bounded MapLibre vector map that enables full cartographic style control and a polished map-first landing state.

## Scope

### In scope

- Replace `leaflet` + `react-leaflet` with `maplibre-gl` + `react-map-gl/maplibre`.
- Preserve the existing map boundary abstraction (`MapPanel` and `MapPanelChromeless`) so feature call-sites stay stable.
- Introduce a custom dark vector style contract (`shared/maps/map-style.ts`).
- Apply UK panning/zoom constraints and a UK-first default view.
- Keep marker popup and keyboard-accessibility behavior at parity with the current baseline.
- Keep map rendering lazy-loaded and inside existing map-chunk performance budget.

### Out of scope

- Bi-directional hover linking and clustering (UX-2).
- Overlay panel redesign and mobile bottom-sheet behavior (UX-3).
- Typography, motion, and site chrome refinements (UX-4/UX-5/UX-6).
- Loading/empty/error polish enhancements (UX-7).

## Decisions

1. Map engine is MapLibre GL JS for vector style control and open-source licensing.
2. React integration uses `react-map-gl` MapLibre entrypoint (`react-map-gl/maplibre`) to minimize bespoke map lifecycle wiring.
3. Migration remains behind existing component boundary:
   - `MapPanelLeaflet` -> `MapPanelMapLibre`
   - `MapPanel` and `MapPanelChromeless` public interfaces unchanged.
4. UK map constraints are enforced via map options and runtime move guards:
   - approximate bounds SW `[49.5, -8.2]`, NE `[61.0, 2.0]`
   - `minZoom` around `5`, `maxZoom` around `17`.
5. Landing state always renders a map canvas (no empty placeholder panel) even before first successful search.
6. Failure behavior must be explicit: if vector style/source fails to load, retain container layout and render actionable fallback messaging instead of blank canvas.

## Frontend Design

### Map Style Design Intent

The map is not a background — it is the primary visual surface of the product. Every style decision must serve one principle: **the map should look like it was designed by a graphic designer, not generated by a cartography API.**

#### Visual DNA

Think architectural blueprint meets dark data dashboard. The map reads as a **precision instrument**: flat, crisp, intentionally sparse, with geometric authority at every zoom level. Street detail is preserved but visually subordinated — the data layer (school markers) owns all visual energy.

Reference touchpoints:

- [The Refugee Project](https://www.therefugeeproject.org/) — monochrome map as editorial canvas.
- [Felt dark style](https://felt.com/) — clean vector with selective detail.
- Dark-mode Apple Maps — geometric roads, flat water, restrained labels.

#### Colour Mapping (Civitas tokens to map features)

The entire map lives inside the navy palette. No feature should break out of it.

| Map feature | Token / hex | Reasoning |
|---|---|---|
| Land fill | `navy-950 → #060a13` | Map background is the darkest canvas — recedes completely. |
| Water fill | `navy-900 → #0c1222` | Barely distinguishable from land. Water is not a highlight. |
| Green space | `navy-900` at 60% opacity | Parks exist but do not compete. Subtle tonal shift only. |
| Motorway/trunk | `navy-600 → #263350` at 80% opacity | Strongest road class. Visible but never bright. |
| A-road | `navy-700 → #1c2840` at 70% opacity | Secondary road presence. |
| Local road | `navy-800 → #141d30` at 50% opacity | Street grid visible, not dominant. Ghost lines. |
| Building fill | `navy-800 → #141d30` | Flat polygons, zero stroke. Emerge quietly at z15+. |
| Building stroke | None | No outlines. Buildings are mass, not wireframe. |
| Rail line | `grey-500 → #64748b` at 40% opacity, dashed | Subtle infrastructure hint. |
| Boundary line | `navy-700 → #1c2840` at 30% opacity | Almost invisible. Context only. |
| Label text | `grey-400 → #94a3b8` at 60% opacity | Muted. Present for wayfinding, never shouty. |
| Label halo | `navy-950 → #060a13` at 80% opacity | Tight, matching background. No bright halo glow. |
| **School marker** | `brand-500 → #8b5cf6` | **The only saturated element on the entire map.** |
| Marker glow | `brand-500` with `0 0 8px 3px` box-shadow | Markers emit light. The map does not. |

This creates a **single visual hierarchy**: navy background → ghost-line geometry → luminous purple data points. Your eye has nowhere to go except the markers.

#### Typography On Map

- Labels use **Space Grotesk** (the app's display typeface) loaded as a MapLibre glyph range.
- If glyph hosting is not feasible in v1, use **Noto Sans** as the closest geometric sans fallback available from standard glyph CDNs.
- Label sizes are deliberately small. Major city names at z8 are 12px. Neighbourhood names at z13 are 10px. Street names at z15+ are 9px.
- All labels are `text-transform: uppercase` with `letter-spacing: 0.08em` — matching the app's mono-label treatment.
- Labels never use bold weight on the map. `font-weight: 500` maximum.

#### Zoom-Level Design Contract

Each zoom tier has a specific visual character. This is the creative contract — not just what data appears, but what the map *feels like* at that scale.

**z5–z8 — National (The Abstract)**
- Land, water, and coastline only.
- Motorways as thin single lines (1px). No casing, no glow.
- Country and major city labels only.
- The map looks like a **minimal data visualization** — a shape with sparse annotation.
- Feel: clean enough to be a poster.

**z9–z11 — Regional (The Network)**
- A-roads appear as thinner ghost lines.
- Borough and district labels emerge.
- Rail lines appear as subtle dashed strokes.
- Admin boundaries as near-invisible hairlines.
- The road network reads as a **geometric mesh** — you see connectivity, not individual streets.
- Feel: London Tube map energy. Schematic, not photographic.

**z12–z14 — City (The Grid)**
- All roads visible. Local roads as the lightest ghost lines.
- Neighbourhood labels.
- Green spaces appear as barely-visible tonal shapes.
- Water features (rivers, canals) gain a faint outline.
- The map reads as a **street grid** — crisp, flat, architectural.
- Feel: urban planning document. Every line has purpose.

**z15–z17 — Street (The Detail)**
- Buildings appear as flat-filled polygons. No outlines, no shadows.
- Street names appear in small uppercase.
- Full detail, but rendered with the same flat, geometric, desaturated discipline.
- Feel: zoomed-in section of an architectural plan. You could frame it.

#### Anti-Patterns (Things the map must never do)

- Never show gradient fills on land or water.
- Never show road casing (outline+fill double-stroke technique).
- Never use tapered line joins or rounded caps on roads.
- Never render terrain hillshade or elevation shading.
- Never use bright colors for any map feature (only markers get saturation).
- Never show feature icons (bus stops, shop icons, fuel stations etc.).
- Never allow label collision to create visual clutter — aggressively hide lower-priority labels.
- Never use different typefaces for different label classes.

#### Tile Source Decision (Blocking)

The style spec above requires OpenMapTiles-schema vector tiles with building, road, water, boundary, and label layers. Recommended resolution path:

| Option | Pros | Cons | Verdict |
|---|---|---|---|
| **Protomaps free CDN** | Free, global CDN, PMTiles format, full OSM data, no API key | Less mature CDN SLA, tile freshness varies | **Recommended for v1.** Zero cost, full style control, can self-host later. |
| MapTiler hosted | Very reliable CDN, good defaults, API key auth | Free tier has request limits, requires key management | Good fallback if Protomaps proves unreliable. |
| Self-hosted PMTiles on S3/R2 | Full control, no third-party dependency | Requires tile generation pipeline, ops overhead | Future option after v1 proves the style. |

Decision: **Protomaps free CDN for UX-1 implementation. MapTiler as explicit fallback provider. Re-evaluate for self-hosting after UX-1 ships.**

#### Style Authoring Path

1. Start from the [Protomaps dark basemap style](https://github.com/protomaps/basemaps) as the structural skeleton — it already targets the Protomaps tile schema.
2. Strip it back to near-zero visual weight using the colour table above.
3. Override label font stacks with Space Grotesk (or Noto Sans fallback).
4. Use [Maputnik](https://maputnik.github.io/) for visual iteration.
5. Export the final style JSON and commit it at `apps/web/src/shared/maps/civitas-dark.json`.
6. The style resolver (`map-style.ts`) loads this JSON, injects any runtime config (tile URL overrides, glyph URLs), and passes it to MapLibre.

## Acceptance Criteria

1. Leaflet stack is fully removed from dependencies and runtime.
2. Search page renders UK-bounded vector map at first load.
3. User cannot pan map outside configured UK bounds.
4. Marker/popup keyboard and screen-reader behavior is preserved.
5. Map-chunk and web performance budgets remain within thresholds.
6. Style/source failures are recoverable and do not leave an unusable map area.
7. Map canvas uses committed Civitas dark style JSON — not a generic third-party theme.
8. School markers are the only saturated visual element on the map surface.
9. Map labels render in Space Grotesk (or approved geometric sans fallback) — never a serif or generic system font.
10. At z12 the map reads as a flat geometric street grid with no road casing, gradients, or terrain shading.

## Risks And Mitigations

- Risk: Map chunk growth reduces room for downstream UX work.
  - Mitigation: re-baseline bundle budgets immediately after UX-1 and track headroom before UX-2/UX-5.
- Risk: vector provider outages or token/config mistakes break map rendering.
  - Mitigation: centralized style resolver with fallback and explicit error UI.
- Risk: migration breaks existing E2E and accessibility assumptions tied to Leaflet CSS classes.
  - Mitigation: replace class-based selectors with role/data-testid selectors and add parity checks.
- Risk: browser/WebGL compatibility regressions.
  - Mitigation: include a tested non-blank fallback panel for unsupported or failed rendering paths.
