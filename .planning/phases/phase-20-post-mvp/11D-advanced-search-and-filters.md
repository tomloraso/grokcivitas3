# 10D - Advanced Search And Filters

## Goal

Expand the search journey beyond postcode and radius with richer shortlist-building controls.

## Scope

- school type and phase filters
- Ofsted or area-context filters where meaningful
- sorting and result refinement behavior

## Guardrails

- Search additions should remain fast and predictable.
- Filter semantics must map to backend-supported query behavior rather than frontend-only filtering of large result sets.

## Acceptance Criteria

- Filter set is defined with clear backend and frontend responsibilities.
- Search performance and interaction quality remain acceptable on desktop and mobile.
