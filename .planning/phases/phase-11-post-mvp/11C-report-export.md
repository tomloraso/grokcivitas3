# 10C - Report Export

## Goal

Add a report-export capability for school or postcode research once the core product surface is stable.

## Scope

- define export targets such as school profile report, compare report, or postcode summary
- define export format and rendering strategy
- reuse existing API contracts and AI-summary rules where appropriate

## Guardrails

- Export content must remain source-backed and clearly dated.
- AI overview content, if included, must carry the same disclaimer and provenance rules as the web product.

## Acceptance Criteria

- Export scope is specific and tied to one user workflow.
- Rendering strategy and supporting backend endpoints are documented.
